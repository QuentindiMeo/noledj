## Exercice 4 — Comparer deux options avec rigueur

### FastAPI

#### Avantages

1. Performance élevée grâce à l'utilisation d'ASGI => bénéfique si l'application évolue vers des besoins plus exigeants en performance.
2. Support natif de la validation des données et de la documentation automatique via OpenAPI => facilite le développement et la maintenance de l'API.
3. Syntaxe moderne et concise => meilleure lisibilité du code.

#### Inconvénients

1. Courbe d'apprentissage pour l'équipe => on risque de perdre du temps sur la V1.
2. Moins de fonctionnalités intégrées pour l'administration et la gestion des utilisateurs => plus de travail pour implémenter ces fonctionnalités.

### Django REST Framework

#### Avantages

1. Forte expérience de l'équipe avec Django => développement plus rapide et meilleure qualité du code dès le départ.
2. Fonctionnalités intégrées pour l'administration et la gestion des utilisateurs => réduction du temps de développement.
3. Large communauté et écosystème mature, offrant de nombreux packages et ressources pour résoudre les problèmes courants et accélérer le développement.

#### Inconvénients

1. Performance moins optimale par rapport à FastAPI => problème si l'application évolue vers des besoins plus exigeants en termes de performance.
2. Moins de flexibilité dans la structure du projet et la personnalisation des composants => nécessite des contournements pour répondre à des besoins spécifiques.

### Recommandation

- Option choisie : Django REST Framework
- Contexte : L'application d'administration interne pour 200 utilisateurs nécessite beaucoup de CRUD et une interface d'administration native.
  L'équipe est composée de 3 ingénieurs, dont 2 connaissent Django mais aucun ne connaît FastAPI. Le délai pour la V1 est de 3 mois, et il n'y a pas de besoin de performance extrême.
- Décision : Nous recommandons d'utiliser Django REST Framework pour le développement de l'API.
- Alternatives : FastAPI a été considéré, mais en raison de la courbe d'apprentissage pour l'équipe et du manque de fonctionnalités intégrées pour l'administration, il n'est pas recommandé dans ce contexte.
- Conséquences : L'utilisation de Django REST Framework permettra un développement plus rapide et une meilleure qualité du code dès le départ, grâce à l'expérience de l'équipe avec Django.
  Les fonctionnalités intégrées pour l'administration et la gestion des utilisateurs réduiront le temps de développement et les efforts nécessaires pour mettre en place ces fonctionnalités.
  Cependant, il faudra surveiller les performances à mesure que l'application évolue, bien que cela ne soit pas un problème immédiat dans le contexte actuel.
